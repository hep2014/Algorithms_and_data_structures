## Описание  
  
В работе реализована программа на языке C, которая:  
  
1. считывает числа из текстового файла в очередь, реализованную на базе связного списка; 
2. выводит исходную последовательность;  
3. сортирует данные методом **Stooge Sort**;  
4. выводит отсортированную последовательность;  
5. измеряет время сортировки;  
6. по желанию пользователя выполняет **наивный поиск подстроки** в строковом представлении отсортированной очереди.  
  
Работа демонстрирует использование:  
- очереди на связном списке;  
- рекурсивной сортировки;  
- обработки файлового ввода;  
- проверки корректности числовых данных;  
- простейшего алгоритма поиска подстроки.  
  
---  
  
## Постановка задачи  
  
Необходимо разработать программу, которая работает с последовательностью чисел из файла.  
  
Программа должна:  
- считать числа из файла в очередь;  
- проверить корректность входных данных;  
- отсортировать элементы очереди;  
- вывести результат сортировки;  
- по запросу пользователя выполнить поиск подстроки среди отсортированных чисел.  
  
---  
  
## Используемые структуры данных  
  
### Очередь на базе связного списка  
  
Очередь состоит из узлов, каждый из которых хранит:  
- число типа `double`;  
- указатель на следующий узел.  
  
Для очереди поддерживаются указатели:  
- `front` — начало очереди;  
- `rear` — конец очереди.  
  
### Основные операции очереди  
  
- `createQueue()` — создание пустой очереди;  
- `enqueue()` — добавление элемента в конец очереди;  
- `dequeue()` — удаление элемента из начала очереди;  
- `queueSize()` — подсчёт числа элементов;  
- `printQueue()` — вывод очереди;  
- `queueToString()` — преобразование содержимого очереди в строку.  
  
---  
  
## Алгоритм сортировки  
  
### Stooge Sort  
  
В программе используется рекурсивный алгоритм **Stooge Sort**.  
  
Принцип работы:  
1. сравниваются первый и последний элементы;  
2. если первый больше последнего, они меняются местами;  
3. если длина фрагмента больше двух элементов, рекурсивно сортируются:  
- первые две трети,  
- последние две трети,  
- снова первые две трети.  
  
Сначала очередь копируется в массив, затем сортировка выполняется над массивом, после чего отсортированные значения записываются обратно в очередь.  

  
---  
  
## Наивный поиск подстроки  
  
После сортировки пользователь может ввести последовательность чисел для поиска.  
  
Программа:  
1. преобразует очередь в строку;  
2. получает шаблон поиска;  
3. ищет точные вхождения шаблона в строке;  
4. выводит найденные совпадения с цветовым выделением.  
  
Поиск выполняется алгоритмом **naive string matching**, то есть прямым посимвольным сравнением шаблона со строкой.  
  
---  
  
## Проверка входных данных  
  
Программа проверяет:  
- существование файла;  
- наличие чисел в файле;  
- корректность каждого токена;  
- ситуацию, когда в файле только одно число;  
- корректность пользовательского ответа (`Да` или `Нет`);  
- корректность чисел, введённых для поиска подстроки.  
  
Если обнаружена ошибка, программа завершает работу с сообщением.  
  
---  
  
## Формат входных данных  
  
Файл должен содержать числа типа `double`, разделённые:  
- пробелами;  
- табуляцией;  
- переводами строк.

## Формат работы программы

### Шаг 1. Запуск программы

После запуска программа запрашивает имя файла:
```
Введите название файла с числами
```

### Шаг 2. Чтение и вывод исходных данных

После успешного чтения выводятся числа до сортировки.

### Шаг 3. Сортировка

Программа сортирует очередь методом Stooge Sort и выводит результат.

### Шаг 4. Выбор поиска подстроки

Пользователь вводит:
- `Да` — если хочет выполнить поиск;
- `Нет` — если хочет завершить программу.

### Шаг 5. Поиск подстроки

Если пользователь выбирает `Да`, программа просит ввести последовательность чисел, разделённых пробелами, и выполняет поиск.

gcc src/main.c -o main

## Обработка ошибок

Программа завершает работу с ошибкой в следующих случаях:
- файл не найден;
- файл пустой или не содержит чисел;
- в файле присутствует некорректный токен;
- в файле только одно число;
- пользователь вводит неверный ответ вместо `Да` или `Нет`;
- шаблон поиска содержит нечисловые значения.

## Сложность алгоритмов

### Stooge Sort

Временная сложность очень высокая и значительно хуже классических алгоритмов сортировки.  
Используется только в учебных целях.

### Наивный поиск подстроки
При длине строки `n` и длине шаблона `m` временная сложность составляет:

```
O((n - m + 1) * m)
```
