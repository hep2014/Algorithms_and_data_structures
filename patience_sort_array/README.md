## Описание  
  
В работе реализована программа на языке C, которая:  
  
1. позволяет выбрать тип массива:  
- статический;  
- динамический;  
2. позволяет выбрать способ ввода данных:  
- из файла;  
- из стандартного ввода;  
3. проверяет корректность введённых данных;  
4. сортирует массив методом **терпеливой сортировки**;  
5. выводит отсортированный результат;  
6. измеряет время выполнения сортировки;  
7. выводит объём памяти, выделенной в процессе работы алгоритма сортировки.  
  
Работа демонстрирует:  
- обработку пользовательского ввода;  
- работу со статическими и динамическими массивами;  
- валидацию числовых данных;  
- реализацию patience sort;  
- базовый анализ времени и памяти.  
  
---  
  
## Постановка задачи  
  
Необходимо разработать программу, которая:  
  
- получает от пользователя параметры запуска;  
- создаёт статический или динамический массив;  
- считывает элементы массива из файла или с клавиатуры;  
- выполняет терпеливую сортировку массива;  
- выводит отсортированный результат;  
- измеряет время выполнения и объём выделенной памяти.  
  
---  
  
## Используемые структуры данных  
  
### Массив чисел  
  
Программа работает с массивом элементов типа `double`.  
  
Поддерживаются два варианта:  
- **статический массив** фиксированного максимального размера `MAX_SIZE`;  
- **динамический массив**, выделяемый через `malloc`.  
  
### Стопки для patience sort  
  
Для реализации терпеливой сортировки используется массив стопок.  
  
Каждая стопка описывается структурой:  
  
- `data` — массив элементов;  
- `size` — текущее количество элементов в стопке.  
  
---  
  
## Алгоритм сортировки  
  
### Терпеливая сортировка  
  
Алгоритм работает следующим образом:  
  
1. элементы исходного массива последовательно раскладываются по стопкам;  
2. для выбора нужной стопки используется бинарный поиск;  
3. если подходящей стопки нет, создаётся новая;  
4. затем минимальные элементы с вершин стопок последовательно извлекаются;  
5. на их основе формируется отсортированный массив.  
  
### Особенности реализации  
  
- каждая стопка хранится как массив;  
- количество выделенной памяти для стопок суммируется в переменной `total_sorting_memory`;  
- сортировка выполняется непосредственно над массивом пользователя.  
  
---  
  
## Проверка входных данных  
  
Программа проверяет:  
- корректность выбора типа массива;  
- корректность выбора способа ввода;  
- корректность размера массива;  
- существование входного файла;  
- достаточность количества чисел;  
- корректность каждого введённого значения.  
  
При обнаружении ошибки программа завершает работу с сообщением.  
  
---  
  
## Сценарий работы программы  
  
### Шаг 1. Выбор типа массива  
  
Пользователь выбирает:  
- `1` — статический массив;  
- `2` — динамический массив.  
  
### Шаг 2. Выбор источника ввода  
  
Пользователь вводит:  
- `1` — если данные будут считываться из файла;  
- `0` — если данные будут вводиться вручную.  
  
### Шаг 3. Ввод размера массива  
  
Пользователь указывает количество элементов массива.  
  
### Шаг 4. Ввод данных  
  
В зависимости от предыдущего выбора данные:  
- считываются из файла;  
- вводятся через стандартный ввод.  
  
### Шаг 5. Подтверждение сортировки  
  
Перед сортировкой программа показывает параметры и содержимое массива, затем просит подтверждение.  
  
Если пользователь вводит `1`, запускается сортировка.  
  
### Шаг 6. Вывод результата  
  
После завершения сортировки программа выводит:  
- отсортированный массив;  
- время сортировки;  
- объём памяти, выделенной под структуру стопок.  
  
---  
  
## Формат входных данных  
  
Числа должны быть представлены в формате `double` и разделяться:  
- пробелами;  
- табуляцией;  
- переводами строк.

## Сборка
```
gcc src/main.c -o main
./main
```

## Обработка ошибок

Программа завершает работу с ошибкой в следующих случаях:
- выбран неверный тип массива;
- выбран неверный способ ввода;
- размер массива меньше либо равен нулю;
- размер массива превышает `5000`;
- файл не найден;
- в файле недостаточно чисел;
- во входных данных встречается некорректный токен;
- пользователь вводит нечисловое значение там, где ожидается число.

---

## Измерение производительности

После выполнения сортировки программа выводит:
- **время сортировки** в секундах;
- **объём памяти**, выделенной под стопки алгоритма.

Это позволяет оценить поведение алгоритма на разных наборах данных. 

---

## Сложность алгоритма

Терпеливая сортировка использует:
- бинарный поиск для размещения элемента по стопкам;
- последовательный поиск минимальной вершины среди стопок на этапе сборки результата.

Фактическая производительность зависит от распределения входных данных и числа стопок.
