## Описание  
  
В рамках работы реализована сортировка методом терпеливой сортировки (Patience Sort) с использованием стека в качестве основной структуры данных и последующей записью результата в красно-чёрное дерево.  
  
Работа направлена на изучение:  
- стековых структур данных;  
- балансированных бинарных деревьев поиска (красно-чёрное дерево);  
- алгоритмов сортировки и их реализации.  
  
---  
  
## Постановка задачи  
  
Необходимо разработать программу, которая:  
  
1. Считывает данные из файла;  
2. Проверяет корректность входных данных;  
3. Выполняет сортировку методом терпеливой сортировки с использованием стека;  
4. Записывает отсортированные элементы в красно-чёрное дерево;  
5. Выводит результат.  
  
---  
  
## Используемые структуры данных  
  
### Стек  
  
Используется для реализации этапа "раскладывания карт" в алгоритме терпеливой сортировки.  
  
Основные операции:  
- `push`  
- `pop`  
- `peek`  
  
---  
  
### Красно-чёрное дерево  
  
Используется для хранения отсортированных элементов.  
  
Свойства:  
- Каждая вершина имеет цвет (красный или чёрный)  
- Корень всегда чёрный  
- Красные вершины не могут иметь красных детей  
- Для любого пути от вершины до листьев число чёрных узлов одинаково  
  
Обеспечивает:  
- O(log n) вставку  
- автоматическую балансировку  
  
---  
  
## Алгоритм  
  
1. Считываются числа из файла  
2. Элементы распределяются по "стопкам" (через стек)  
3. Формируется последовательность  
4. Элементы вставляются в красно-чёрное дерево  
5. Выполняется обход дерева для получения отсортированного результата  
  
---

## Сборка и запуск

```
gcc src/main.c -o main
./main
```

## Формат входных данных
Файл должен содержать:
- целые числа
- разделённые пробелами или переводами строки

Пример:
```
5 3 8 1 2
```

## Обработка ошибок

Программа обрабатывает:
- отсутствие файла
- некорректные данные
- пустой файл

---

## Особенности реализации

- Проверка входных данных производится до начала сортировки
- Используется динамическая память
- Реализована защита от некорректного ввода
- Красно-чёрное дерево используется для эффективной финальной сортировки

## Сложность алгоритма

|Этап|Сложность|
|---|---|
|Разбиение на стопки|O(n log n)|
|Вставка в RB-дерево|O(n log n)|
|Итог|O(n log n)|
